# /project:create-page

新規ページを作成します。

## 入力

$ARGUMENTS

## 手順

以下の手順に従って新規ページを作成してください。

### Step 1: 引数の解析

`$ARGUMENTS` が空、または何を作るか不明な場合は、ユーザーに以下を質問してください:

- どんなページを作りたいか（機能の説明）
- ページタイトル（英語、例: "Particle System"）
- 簡単な日本語の説明（例: "パーティクルシステム"）

`$ARGUMENTS` に十分な情報がある場合は、以下を決定してください:

1. **ページタイトル**（英語、スペース区切り。例: "Particle System"）
2. **ディレクトリ名**（ケバブケース。例: "particle-system"）
3. **日本語の説明**（ナビゲーションとドキュメント用。例: "パーティクルシステム"）

決定したら、ユーザーに確認を取ってから次に進んでください。

### Step 2: 重複チェック

```bash
ls public/
```

決定したディレクトリ名が既存のページと重複していないか確認してください。
重複している場合はユーザーに別名を提案してください。

### Step 3: ブランチ作成

main ブランチから新しいブランチを作成します。

```bash
git checkout main
git pull origin main
git checkout -b feat/<ディレクトリ名>
```

### Step 4: agent-loop でページを実装

`/general:agent-loop` スキルを呼び出して、以下のタスクを実行してください。

タスク説明として以下を渡してください:

---

**タスク: 新規ページ「<ページタイトル>」の作成**

以下の作業を **1 スプリント** で完結させてください。

#### 4-1. テンプレートのコピー

`public/template/` の内容を `public/<ディレクトリ名>/` にコピーしてください。
コピー対象: `index.html`, `main.js`, `style.css`

#### 4-2. ページの実装

コピーした各ファイルを以下の要件で実装してください。

**ページの説明**: <ユーザーの指示内容>

**index.html**:
- `<title>` を「<ページタイトル> | atelier.yuske.app」に変更
- `<main>` 内に必要な HTML 構造を追加

**main.js**:
- ページの機能を実装
- 先頭に `// @ts-check` を維持

**style.css**:
- box-sizing リセットは維持
- `px` は使用せず相対単位（`rem`, `em`, `%` 等）を使用
- 細線は `0.0625rem`（= 1px 相当）を使用

**コーディング指針**（`~/.claude/tmp/yuske-nakajima/atelier.yuske.app/heart/` を参照）:
- `const` 優先、`let` は最小限に
- early return でネストを減らす
- 5行超の処理ブロックには名前付き関数を使う
- 三項演算子は適度に（5段以上のチェーンは関数化）
- IIFE 禁止、名前付き関数に切り出す

#### 4-3. public/index.html にリンクを追加

`public/index.html` の `<nav> > <ul>` の末尾（`</ul>` の直前）に以下を追加:

```html
<li>
  <a href="/<ディレクトリ名>/">
    <ページタイトル>
    <span class="nav-description"><日本語の説明></span>
  </a>
</li>
```

#### 4-4. docs/README.md のページ一覧を更新

`docs/README.md` の「ページ一覧」テーブルの末尾に以下を追加:

```markdown
| `/<ディレクトリ名>/` | <ページタイトル>（<日本語の説明>） |
```

#### 完了条件

- [ ] `public/<ディレクトリ名>/` に `index.html`, `main.js`, `style.css` が存在する
- [ ] ページタイトルが正しく設定されている
- [ ] ページの機能が実装されている
- [ ] `npm run check` がエラーなしで通る
- [ ] `public/index.html` にリンクが追加されている
- [ ] `docs/README.md` にページ情報が追加されている

#### 評価方法

- `npm run check` でリント・フォーマットチェック
- 各ファイルの存在確認
- index.html と docs/README.md の更新確認

---

### Step 5: 完了報告

agent-loop が完了したら、以下をユーザーに報告してください:

- 作成されたページのパス
- ブランチ名
- コミット一覧
- 次のアクション（PR 作成を提案）
